WIMPS

Below is useful information for each service WIMPS may have installed. This is a rough draft and will be expanded in the future

The initial landing page for the webportal can be changed to your liking. There is an example in %homepath%\wimps\installers\webportal\example_landing.html. This example displayes the jellyfin dashboard in the left, with sonarr and radarr filling out the rest of the page. You can rename example_landing to landing to replace the default page.

General Info

There now is a install auto start script that runs to install the below services to the auto start folder (except JFA-GO, use the tray icon to set auto start).

The below will be removed when re-writing the WIMPS landing page documentation.

Some of these services will not auto start on computer boot up; Jellyseerr, Caddy, DuckDNS updater, JFA-GO

You can use EventGhost, if installed with WIMPS, to auto start services on boot. See below in EventGhost for examples.

You can manually create shortcuts for those services and put them in %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup

jellyseerr : %homepath%/wimps/installers/arr_apps/jsrun.bat

Caddy : %homepath%/cadddy/caddyrun.bat

DuckDNS updater : %HOMEPATH%\duckdns\Duckdns.exe

JFA-GO : %HOMEPATH%\jfa

Some services do not load in the webportal iframe, they will open in a new tab or window. If you're having an issue with a service displaying in the iframe see the below service for more details.

You can edit portal.html loacated at %homepath%\wimps\installers\webportal\portal.html, and add services to the iframe exempt list, if you prefer that behavior or cant get a service to load in the iframe.

At line 110

var iframeExceptions = ['Jellyseerr', 'Threadfin'];
can be expanded to include other services. Such as
var iframeExceptions = ['Jellyseerr', 'Threadfin', 'qBittorrent'];

qBittorrent

To properly get qBittorrent to display in the iframe

qBittorrent > tools > options > webui > uncheck clickjacking protection and if using the webportal from a different machine then uncheck Enable Cross-Site Request Forgery Protection

Change the port to 8081, as that is what the webportal is looking for

SABNZB

To get sabnzb to display in the webportal iframe, acces the webui in a seperate browser tab or window:

sabnzb config (gear icon)> special > uncheck x_frame_options

EventGhost

Open EventGhost and in the right hand pane where it says AutoStart :

Right click AutoStart > add action > System > Start Application > locate the .bat or exe

Make sure to set the working directory section to the same directory the .bat or .exe is
